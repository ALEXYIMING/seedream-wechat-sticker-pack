# Seedream 微信表情包工作流细节

## 输出目录

所有目录都创建在当前项目目录下：

```text
原始目录/      Seedream 生成原图，平铺，不按日期分层
主表情240/     240×240 PNG 主表情
配套图/        banner/cover/icon/reward guide/reward thanks
文案/          album.md 和 文案.txt
打包/          zip 和 summary.json
```

## 推荐步骤

1. 先生成 1-3 张样张确认风格。
2. 整理 16 条 prompt 到 JSON 或 Markdown 文件。
3. 执行 `scripts/seedream_wechat_pack.py`。
4. 人眼检查中文文字和 240×240 可读性。
5. 如果单张不合格，重生该张原图，再用 `--skip-generate` 重新后处理和打包。
6. 如果用户不满意默认配套图，或给了旧专辑物料参考，不要继续堆叠主表情；用参考图做 image-to-image 重新设计 banner/赞赏引导/赞赏致谢，再本地叠准确中文并重新打包。

## Prompt 文件格式

### JSON

```json
[
  {"id": "01", "title": "你一出现我就掉线", "prompt": "完整提示词..."}
]
```

也支持：

```json
{"stickers": [{"id": "01", "title": "...", "prompt": "..."}]}
```

### Markdown

```markdown
01｜你一出现我就掉线
提示词：
完整提示词...

02｜下一张
提示词：
完整提示词...
```

## 透明模式选择

- `keep`：保留背景，适合 8Bit 文字图、强背景撞色图。
- `remove`：去除背景，适合角色/物件表情；采用“采样整圈边缘 + 近角落颜色 + 只删除边缘连通背景块”的抠图方式，避免 Seedream 渐变/晕影背景用简单四角阈值去不干净。
- `auto`：仅在背景很浅时尝试去除背景。

## 配套图精品重设计

当用户说配套图不好，或给出旧项目的 `专辑物料/` 作为参考时，要把 `banner`、`赞赏引导图`、`赞赏致谢图` 当作独立专辑视觉重新设计，不要只把主表情拼贴或堆砌在一起。

推荐做法：

1. 读取参考项目的 `banner.png`、`赞赏引导图.png`、`赞赏致谢图.png`，只学习色彩、构图层级、完整度和专辑物料气质。
2. 用 Seedream `image-to-image` 生成独立设计稿，提示词明确“不要拼贴已有表情/不要堆缩略图/不要文字/不要水印”。
3. 设计主体应是全新元素：复古街机招牌、投币恋爱机、糖果售卖机、爱心宝箱、像素奖杯、恋爱存档机等。
4. 让生成图预留标题牌、文案牌或按钮牌，中文由本地 PIL 叠加，保证文字正确。
5. Seedream API 对直接生成微信规格小图有限制，先高分辨率生成再裁切/缩放：
   - `banner`：例如 `2560x1440` → `750×400`
   - `赞赏引导图`：例如 `2304x1720` → `750×560`
   - `赞赏致谢图`：`2048x2048` → `750×750`
6. 生成后更新 `配套图/` 中标准英文文件名和必要中文别名，再重新打包 zip。

## 失败处理

| 问题 | 处理 |
|---|---|
| 中文错字 | 修改该张 prompt，单张重生 |
| 240×240 后看不清 | 增大字重、减少字数、重生 |
| 主体太小 | prompt 加“主体占画面 85%-90%” |
| 风格不统一 | 固定统一风格前缀和色板 |
| 透明底有背景残留 | 不要只用四角/单色阈值；用边缘连通背景色抠图重新处理该张，再重新打包 |
| 配套图不满意 | 不要只本地重排版；若用户给参考或要求更好，应使用 Seedream image-to-image 重新设计独立专辑物料，再本地叠准确中文 |

## 注意

Seedream 对中文文字生成不保证 100% 准确。文字型表情包最稳方案是：AI 生成背景和装饰，本地脚本叠加文字。当前脚本先支持“AI 直接出整图”的流程，后续可以加本地像素字排版模式。
