# Seedream 微信表情包全套物料编排 Skill

这个 skill 用来把一组表情包创意或提示词，编排成 **微信表情开放平台可用的一整套上架物料**。

它不是单纯的生图工具，而是一个组合型工作流：

1. 调用火山方舟 Seedream 生成无水印原图。
2. 把原图处理成微信主表情规格。
3. 自动生成封面、图标、横幅、赞赏引导图、赞赏致谢图。
4. 生成专辑文案和表情名称清单。
5. 把所有物料打包成 zip，方便上传或交付。

## 适用场景

适合用户提出这些需求时使用：

- “做一套微信表情”
- “用 Seedream 生成微信表情包”
- “生成 16 张/24 张表情并做成微信上架物料”
- “把这些提示词做成微信表情包”
- “需要主表情、横幅、封面、图标、赞赏图和文案一起打包”

不适合的场景：

- 只生成单张图片预览。
- 只做普通 AI 生图，不需要微信规格处理。
- Telegram、Discord、iMessage 等非微信表情平台物料。
- 需要人工精修复杂中文字形的最终商业稿。

## 这个 skill 会做什么

### 1. 生成原图

使用 `byted-ark-seedream-skill/scripts/generate.js` 作为唯一生图引擎，默认关闭水印。

原图会平铺保存到：

```text
原始目录/
```

不会按日期创建多层目录，也不会一张图一个文件夹。

### 2. 处理主表情

把原图统一处理为微信主表情规格：

```text
主表情240/
├── 01_标题.png
├── 02_标题.png
└── ...
```

主表情规格：

- PNG 格式。
- 240×240 像素。
- 中心裁切为正方形后缩放，不拉伸变形。
- 支持保留背景、去除纯色背景或自动判断背景。

### 3. 生成配套图

自动从主表情本地排版生成 5 张微信配套图：

```text
配套图/
├── banner_750x400.png
├── cover_240.png
├── icon_50.png
├── reward_guide_750x560.png
└── reward_thanks_750x750.png
```

配套图优先使用本地排版，而不是重新 AI 生图，这样可以降低中文错字、风格跑偏和表情不一致的问题。

### 4. 生成文案

生成微信表情专辑需要的本地文案：

```text
文案/
├── album.md
└── 文案.txt
```

内容包括：

- 专辑名称
- 专辑介绍
- 赞赏引导语
- 赞赏致谢语
- 每张表情名称

### 5. 打包交付

最终生成 zip：

```text
打包/<album-name>_微信表情物料.zip
```

zip 内包含：

- `主表情240/`
- `配套图/`
- `文案/`

## 输出目录结构

默认所有产物都输出在当前项目目录下：

```text
<project>/
├── 原始目录/          # Seedream 原图，平铺保存
├── 主表情240/         # 16/24 张 240×240 PNG 主表情
├── 配套图/            # 5 张微信配套图
├── 文案/              # album.md / 文案.txt
└── 打包/              # 微信表情物料 zip
```

## 推荐工作流

### 阶段 0：先出样张

新主题、新风格不要直接批量生成。建议先生成 1-3 张样张，检查：

- 中文文字是否正确、可读。
- 缩小到 240×240 后是否仍清楚。
- 是否无水印。
- 风格是否统一。
- 主体大小和透明底效果是否符合预期。

### 阶段 1：整理提示词文件

把 16/24 条表情整理成 JSON：

```json
[
  {
    "id": "01",
    "title": "你一出现我就掉线",
    "prompt": "完整提示词..."
  },
  {
    "id": "02",
    "title": "下一张",
    "prompt": "完整提示词..."
  }
]
```

也支持这种结构：

```json
{
  "stickers": [
    {
      "id": "01",
      "title": "你一出现我就掉线",
      "prompt": "完整提示词..."
    }
  ]
}
```

Markdown 格式也支持：

```markdown
01｜你一出现我就掉线
提示词：
完整提示词...

02｜下一张
提示词：
完整提示词...
```

### 阶段 2：批量生成、处理、打包

示例命令：

```bash
python3 /Users/alexyiming/.claude/skills/seedream-wechat-sticker-pack/scripts/seedream_wechat_pack.py \
  --prompt-file ./prompts_8bit_love.json \
  --theme "8Bit 土味情话" \
  --album-name "像素情话" \
  --intro "一套复古街机像素风土味情话表情包，高对比撞色，直球心动。" \
  --tip "喜欢就给点电" \
  --transparent-mode keep
```

## 常用参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `--project-dir` | 当前目录 | 所有输出目录所在项目目录 |
| `--prompt-file` | 必填 | JSON 或 Markdown 提示词文件 |
| `--theme` | 自动 | 主题名称，用于文案和排版 |
| `--album-name` | 自动 | 微信表情专辑名称，建议不超过 8 个字 |
| `--intro` | 自动 | 专辑介绍，建议不超过 80 字 |
| `--tip` | 自动 | 赞赏引导语 |
| `--thanks` | 自动 | 赞赏致谢语 |
| `--size` | `2K` | Seedream 生成尺寸 |
| `--response-format` | `png` | 生成图片格式 |
| `--optimize` | `true` | 是否让 Seedream 优化提示词 |
| `--skip-generate` | `false` | 跳过生图，直接处理 `原始目录/` 已有图片 |
| `--transparent-mode` | `keep` | 背景处理模式：`keep` / `remove` / `auto` |

## 背景处理模式

| 模式 | 适合场景 | 效果 |
|---|---|---|
| `keep` | 文字图、强背景设计、撞色像素图 | 保留原背景 |
| `remove` | 角色、图标、道具、白底或纯色底图片 | 尝试去除纯色背景，输出透明 PNG |
| `auto` | 不确定是否应该透明 | 仅在背景很浅时尝试去除 |

如果用户明确要求“透明底”“去除背景”“240×240 透明 PNG”，应使用：

```bash
--transparent-mode remove
```

## 质量检查

每次执行完成后，应报告：

- 原图数量。
- 主表情数量。
- 配套图数量。
- 是否有失败项。
- zip 路径。
- 关键本地路径。

如果出现问题：

| 问题 | 建议处理 |
|---|---|
| 中文错字 | 修改该张 prompt，单张重生 |
| 240×240 后看不清 | 增大字重、减少文字、让主体占画面 85%-90% |
| 主体太小 | prompt 加“主体居中，占画面 85%-90%” |
| 风格不统一 | 固定统一风格前缀、固定色板和构图规则 |
| 配套图不满意 | 优先本地重排版，不建议 AI 重画中文 |

## 注意事项

Seedream 对中文文字生成不能保证 100% 准确。文字型表情包建议先做样张并人工确认，必要时可以改为“AI 生成装饰元素，本地脚本叠加文字”的方案。
